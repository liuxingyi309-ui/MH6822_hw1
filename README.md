## Project

**HSBC PEP Screening: US vs UK**

I worked on HSBC PEP Screening: US vs UK. The main idea is to show how a PEP screening tool could apply different AML/KYC logic in the United States and the United Kingdom, instead of using one global rule for every case.

## Files

- `Task1_selection_research.md`  
  Selection of HSBC, the AML/KYC PEP screening domain, and US/UK jurisdiction rationale.

- `Task2_values_audit.md`  
  Values audit answering the four required questions.

- `Task3_design.md`  
  Main tool design explanation for the working prototype.

- `Task3_tool.py`  
  Python prototype for jurisdiction-aware PEP screening.

- `data/Task3_data.csv`  
  Synthetic customer data used by the prototype.

- `Task3_results.csv`  
  Output generated by the prototype.

- `Task3_summary.md`  
  One-page written summary of the design choices.

- `Task3_governance_stub.md`  
  Governance stub covering assumptions, boundaries, failure points, and human review.

## Running the prototype

From the project folder, run:

```bash
python Task3_tool.py
```

The script reads `data/Task3_data.csv` and writes `Task3_results.csv`.

## Data note

The customer data used in this project is synthetic. No real customer data is used.
